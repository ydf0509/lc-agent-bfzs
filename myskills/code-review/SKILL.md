# 代码审查技能

当用户要求你审查代码时，请按照以下步骤进行：

## 审查要点

1. **代码风格**: 检查命名规范、缩进、代码组织
2. **潜在 Bug**: 空指针、类型错误、边界条件
3. **性能**: 不必要的循环、内存泄漏风险
4. **安全性**: SQL 注入、XSS、敏感信息泄露
5. **可维护性**: 代码重复、函数过长、耦合度

## 输出格式

请按以下格式输出审查结果：

```
## 审查总结
- 整体评价（优/良/需改进）

## 发现的问题
1. [严重程度: 高/中/低] 问题描述
   - 位置: 文件名:行号
   - 建议: 修改方案

## 亮点
- 值得称赞的设计或实现
```
